# 06_product — デザインコンセプト・テーマカラー指示

このファイルは `06_product` 配下の静的 HTML（臨床ツール窓口・各計算機・チェックリスト等）を作成・修正する際の**唯一のデザイン参照元**です。  
新規ページを追加する場合も、以下のコンセプト・配色・UI原則を必ず踏襲してください。

このデザイン方針は、**紙の上に色面を重ねたような幾何学的ポスター表現**をベースにしています。  
参考イメージ: [参考画像](https://www.genspark.ai/api/files/s/cIcZhuVW)

---

## 1. プロダクトコンセプト

| 項目 | 内容 |
|------|------|
| **役割** | 診療・教育を支える**臨床向けミニツール群**。最優先は正確さ・可読性・操作の速さ。 |
| **利用シーン** | 病棟、外来、カンファレンス前後などの**短時間・高集中**の利用。小さな画面でも判断しやすいこと。 |
| **トーン** | 医療資料の信頼感を保ちながら、**幾何学的でシャープ、構成美のある画面**にする。 |
| **目指す質感** | フラットな色面、明快なレイヤー、強いタイポグラフィ、控えめな影。**紙を重ねたような整理された緊張感**を持たせる。 |
| **非目標** | AI生成っぽい多重枠、意味のない装飾線、無駄なイラスト、過剰な未来感、やわらかすぎるUI、派手すぎるアニメーション。 |

**一言で:**  
ベージュの紙地の上に、**黄緑〜緑・ティール・水色の幾何学色面を大胆に重ねる**（注意はオレンジ系で限定）。  
装飾で魅せるのではなく、**構図・色面・文字の強さ**で成立させる。

---

## 2. デザイン原則

### 2.1 見た目の基本姿勢

- 情報は装飾ではなく、**大きな面と明快な階層**で見せる
- デザインの印象は、色よりも**構図・整列・レイヤー・余白**で作る
- 「凝っている感じ」ではなく、**少ない要素で強く見える構成**を優先する
- すべての要素は、**意味のある塊**として配置する
- 画面全体をUI部品で埋めず、**背景の余白も構図の一部**として扱う

### 2.2 禁止事項

以下は原則として使わないこと。

- 不必要なイラスト
- 意味のない背景模様
- AI生成っぽい発光、ネオン、装飾線
- 枠を何重にも重ねる表現
- ふわっと大きく広がる影
- 過剰なグラデーション
- 全部を丸くするUI
- 装飾目的だけのアイコン
- 情報整理に寄与しない色の追加
- 情報量の少ない場面でカードを乱立させること

### 2.3 推奨する表現

- **大きな長方形・角丸長方形・細い帯・円弧**を主役にする
- 形は少数に絞り、**画面の端で見切らせる**ことで構図を作る
- 強調は、発光や装飾でなく**前後の重なりと色面の対比**で行う
- ボタンや入力欄も、柔らかい箱ではなく**幾何学的な操作部品**として扱う
- 情報のまとまりは、カードを増やすより**大きな背景面と明快な見出し**で整理する
- 1画面内の要素数を抑え、**大きな形と強い文字**で印象を作る

## 3. カラーパレット（必ずこの CSS を使用すること）

各 HTML の `:root` に、**以下をそのまま**置くこと。  
変数名・HEX は変更しない。値を変更する場合はこのファイルを更新し、全ページに反映する。

```css
:root {
  --color-bg:      #E2DFCC;
  --color-dark:    #455F51;
  --color-accent1: #99CB38;
  --color-accent2: #63A537;
  --color-accent3: #37A76F;
  --color-accent4: #44C1A3;
  --color-accent5: #4EB3CF;
  --color-accent6: #51C3F9;
  --color-warning: #EE7B08;
  --color-white:   #FFFFFF;
}
```

### 3.1 色の考え方

このプロダクトでは、色は「優しさ」よりも**構図の強さと機能の識別**に使う。

| トークン | 主な用途 |
|----------|----------|
| `--color-bg` | ページ背景。紙のような静かな土台。 |
| `--color-dark` | 本文、見出し、強いラベル、主要線・帯の締め。 |
| `--color-white` | 入力欄、情報面、数値表示面などの作業領域。 |
| `--color-accent1` | 明るい黄緑のベース面、広めの背景パネル。 |
| `--color-accent2` | accent1 より締まった緑の面、補助パネル。 |
| `--color-accent3` | 主役の緑の強調面、主ボタン、肯定的なアクセント。 |
| `--color-accent4` | ティール系の補助面、ホバー、別レイヤー。 |
| `--color-accent5` | 水色〜シアン、角や細部の差し色。 |
| `--color-accent6` | 明るいスカイブルー、細帯、フォーカス、上層のハイライト。 |
| `--color-warning` | 注意、確認、入力不備、スクリーニング陽性など。 |

### 3.2 アクセント色の運用ルール

- 画面の主役は **`accent1` または `accent3` のどちらか一方**にする
- `accent6` は**明るい差し色・細帯・フォーカス**などに少量だけ使う（画面の締めは `--color-dark`）
- `accent5` は大きく使いすぎず、**端や角のアクセント**として使う
- 1画面で色数を増やしすぎない。**3〜4色で成立させる**
- 色面は小さく散らさず、**大きく使う**

### 3.3 派生色の作り方

新しい色トークンは追加せず、既存トークンから派生させる。

- **弱い補足テキスト**
  ```css
  color: color-mix(in srgb, var(--color-dark) 72%, var(--color-bg));
  ```

- **区切り線・カード枠**
  ```css
  border-color: color-mix(in srgb, var(--color-dark) 16%, var(--color-white));
  ```

- **控えめな影**
  ```css
  box-shadow: 0 10px 24px color-mix(in srgb, var(--color-dark) 16%, transparent);
  ```

**影はUIらしい立体感ではなく、紙を重ねたような前後関係を見せるためだけに使うこと。**

## 4. 造形ルール

### 4.1 形状

- 基本形は**長方形**
- 使用する形は以下に限定する
  - 長方形
  - 角丸長方形
  - 細い横帯
  - 円弧または円の一部
- 角丸は一部の大きな面だけに使う
- 小要素まで丸くしすぎない
- 「全部同じ角丸」にせず、**直角と曲線を混在**させることで緊張感を保つ

### 4.2 線

- 情報の整理は影より**面の境界と整列**で行う
- 線は細く最小限にする
- 枠で囲いすぎない
- 枠が必要なときも、**1要素1境界**を原則とする
- 強い区切りは、線よりも**濃色の細帯**で表現してよい

### 4.3 余白

- 余白は単なる空きではなく、**色面を引き立てる背景**として扱う
- 要素を中央に詰め込みすぎない
- 背景の見える面積をしっかり残す
- 画面の端に図形を見切れさせる場合は、**余白とのバランス**を優先する
- “情報を並べる”より“構図を作る”意識で空ける

## 5. レイアウト原則

### 5.1 グリッド

- レイアウトは**幾何学的な整列**を前提にする
- 左端、見出し、本文、ボタン列の基準線を揃える
- ただし、背景の色面は**少し大きくはみ出す構成**を許容する
- 情報ブロックは整列させ、装飾面はその外側で構図を作る
- UIとしての整列と、ポスター的な大胆さを両立させる

### 5.2 情報の優先順位

1. ページ名・機能名
2. 入力すべき内容
3. 結果
4. 注意点
5. 補足説明

見た目の演出より、**視線が迷わず流れること**を優先する。

### 5.3 カードの扱い

- 何でもカード化しない
- カードは「白い作業面」としてのみ使う
- 画面全体はカードの集合ではなく、**大きな背景面の上に必要な作業面を載せる**考え方にする
- カードで細かく区切るより、**大きな色面と見出しで場を作る**

## 6. タイポグラフィ

- **フォント:**  
  `"Hiragino Sans", "Hiragino Kaku Gothic ProN", "Noto Sans JP", sans-serif`

- **本文サイズ:**  
  16px 前後を基準

- **行間:**  
  `line-height: 1.5`〜`1.6`

- **見出し:**  
  太く、強く、大きく使う  
  情報の主役は、装飾よりも**文字の強度**で作る

- **数字:**  
  スコア・計算結果・単位は、本文より強く見せる  
  太さ、サイズ、余白で差をつける

- **見出しの扱い:**  
  1行で強く見せることを優先し、無理に小さく収めない  
  必要なら改行してもよいが、**塊感**を失わないこと

## 7. コンポーネント指針

### 7.1 ボタン

- 主ボタンは**アクセント緑（`accent3` など）または濃色（`color-dark`）**で強く見せる
- 補助ボタンは白地または淡色で抑える
- ボタンは丸すぎず、**やや硬い形**にする
- サイズをむやみに大きくせず、文字の強さで押しやすさを作る
- 影は最小限にとどめる

### 7.2 入力欄

- 入力欄は基本的に白
- 背景の色面とは明確に分離する
- ラベルは近接配置し、対応関係を明快にする
- 入力欄まわりに余計な装飾を入れない
- フォーカスは色で明示しつつ、過剰に光らせない

### 7.3 結果表示

- 結果は、画面内で最も目立つ情報として置く
- 数値は大きく、短く、明快に見せる
- 結果の箱は白地でもよいが、**背景の色面との対比**で印象を作る
- 色だけで意味を伝えず、ラベルや注記も添える
- 派手な演出は不要。**即読性**を優先する

### 7.4 警告・注意

- `--color-warning` は限定的に使う
- 多用すると画面が崩れるため、本当に必要な箇所だけに使う
- 注意表示は、色だけでなく文言と配置でも伝える
- 警告をカードで囲いすぎない

---

## 8. フォーカス・ホバー・状態変化

### 8.1 基本方針

状態変化は演出ではなく、**レイヤーの前後差と操作の確信**として設計する。

### 8.2 推奨

```css
:focus-visible {
  outline: 2px solid var(--color-accent6);
  outline-offset: 2px;
}
```

- ホバーは色を1段階変える程度にする
- 拡大、跳ねる、強い発光は使わない
- アニメーションは短く、静かに、必要最小限にする
- 変化は「派手さ」よりも「気づきやすさ」を優先する

## 9. このスタイル特有の構図ルール

### 9.1 背景構成

- 背景は単なる無地ではなく、**大きな色面を数枚重ねる構成**を基本とする
- 主役の面を1つ、補助の面を2〜3つまでに抑える
- 面は揃えすぎず、少しずらして重ねる
- 1本だけ細い濃色の帯を入れると画面が締まる

### 9.2 見切れの使い方

- 円弧や大きな長方形は、画面外にはみ出してよい
- ただし、見切れは**1〜2箇所まで**
- すべての要素を見せ切らず、**構図の途中を切り取った感じ**を作る
- 見切れた形が情報を邪魔しないことを優先する

### 9.3 レイヤーの使い方

- 面の重なりは3層程度まで
- 多層にしすぎない
- 前後関係は、色・位置・影で明快にする
- 立体物っぽくしすぎず、**紙を重ねた平面的な奥行き**に留める

## 10. 再現性のためのルール

このプロダクトでは、1ページごとの思いつきより、**誰が作っても同じ空気感になること**を重視する。

### 必須ルール

1. 独自の装飾モチーフを足さない
2. 独自の色を直書きしない
3. UI部品を全部丸くしない
4. 大きな色面と強い文字を基本にする
5. 枠よりも構図で整理する
6. 迷ったら「要素を減らす」方向を選ぶ
7. 装飾を足すより、背景面・余白・整列を見直す

### 判断基準

追加した要素が以下のどれにも当てはまらない場合は、削除を優先する。

- 読みやすくなる
- 操作しやすくなる
- 情報の優先順位が明確になる
- 構図が強くなる
- 他ページでも再利用できる

## 11. 実装チェックリスト（新規 HTML 用）

1. `:root` に **本ファイル記載のカラーパレットをそのまま**置く
2. `body` は原則 `background: var(--color-bg); color: var(--color-dark);`
3. 背景には必要に応じて**大きな幾何学色面**を配置する
4. 主役の色面は 1つ、補助色面は 2〜3つまでに抑える
5. UI部品は白地ベースで、背景面と分離する
6. 主見出しは太く強く扱う
7. `accent6` は差し色・細帯・フォーカスとして少量だけ使う（締めは `color-dark`）
8. 影は紙の重なり感を出す最小限に留める
9. 不要なイラスト・装飾・多重枠を入れない
10. 幾何学的で、シャープで、再現性の高い画面にする
11. 窓口リンクを追加する場合は `portal/index.html` のリストも更新する

## 12. 変更の扱い

- **パレットやデザイン原則を変更する場合**  
  まずこのファイルを更新し、その後 `portal/index.html` と各HTMLへ反映する

- **表現に迷った場合**  
  「より派手」ではなく、**より少ない要素で強く見える**方向を選ぶ

- **新しい装飾案を追加したい場合**  
  単ページの思いつきで入れず、全体で再現可能かを確認してから採用する
