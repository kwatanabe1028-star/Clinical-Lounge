# 06_product — コンセプト・テーマカラー指示

このファイルは `06_product` 配下の静的 HTML（臨床ツール窓口・各計算機・チェックリスト等）を作る・直すときの**単一の参照元**です。新規ページを追加する際は、ここで定義したコンセプトとカラーを踏襲してください。

---

## 1. プロダクトコンセプト

| 項目 | 内容 |
|------|------|
| **役割** | 診療・教育の補助として使う**臨床向けミニツール**の集合。派手さより正確さと読みやすさ。 |
| **利用シーン** | 病棟・外来・カンファレンス前後の短時間利用。画面は小さくても可読であること。 |
| **トーン** | 落ち着いた医療文書調。煽り・娯楽要素は入れない。見出しと操作は明確に。 |
| **非目標** | ブランド広告のような強い個性、ダークモード専用デザイン（必要なら別節で拡張）。 |

**一言で:** ベージュグリーンの背景（`--color-bg`）と深緑の字（`--color-dark`）で院内資料のような落ち着きを保ち、`--color-accent1`〜`--color-accent6` で段階・種類を表すデジタル操作感を足す。

---

## 2. カラーパレット（必ずこの CSS を使用すること）

各 HTML の `:root` に、**以下をそのまま**置くこと。変数名・HEX を変更しない（値の変更はこのファイルを更新してから全ページに反映する）。

```css
:root {
  --color-bg:      #E2DFCC;
  --color-dark:    #455F51;
  --color-accent1: #99CB38;
  --color-accent2: #63A537;
  --color-accent3: #37A76F;
  --color-accent4: #44C1A3;
  --color-accent5: #4EB3CF;
  --color-accent6: #51C3F9;
  --color-warning: #EE7B08;
  --color-white:   #FFFFFF;
}
```

### 2.1 用途の目安（トークンは増やさず、意味を共有する）

| トークン | 主な用途 |
|----------|----------|
| `--color-bg` | ページ背景。 |
| `--color-dark` | 本文・見出し・強い境界に近い線。 |
| `--color-white` | カード・パネル・入力欄の下地。 |
| `--color-accent1` … `--color-accent6` | 黄緑→緑→ティール→水色の階段。**主ボタン・リンク・フォーカス**は `accent3`〜`accent5` あたりを優先し、ホバーは一段濃い方（例: `accent3` → `accent2`）に寄せる。グラフ・複数系列では 1→6 の順で色分けしてよい。 |
| `--color-warning` | 注意喚起・要確認・入力エラーなど（本文は読める組み合わせにする）。 |

### 2.2 境界線・影・補足テキスト（パレット外の色を増やさない）

`:root` に変数を足さず、`color-mix` などで既存トークンから派生させる。

- **補足・フッターなどやや弱い字:**  
  `color: color-mix(in srgb, var(--color-dark) 70%, var(--color-bg));`（必要なら比率だけ調整）
- **区切り線・カード枠:**  
  `border-color: color-mix(in srgb, var(--color-dark) 20%, var(--color-white));` など
- **影:**  
  `box-shadow: 0 1px 3px color-mix(in srgb, var(--color-dark) 10%, transparent);` や、アクセントに寄せた `color-mix(..., var(--color-accent3), transparent)` など

## 3. タイポグラフィとレイアウト（推奨）

- **フォント:** `"Hiragino Sans", "Hiragino Kaku Gothic ProN", "Noto Sans JP", sans-serif`
- **本文サイズ:** 16px 前後を基準。高齢者向けツールは 1 段階大きくしてよいが、トーンは維持する。
- **行間:** `line-height: 1.5`〜`1.6`
- **角丸:** カード・ボタンは `8px`〜`12px` 程度で統一。
- **コントラスト:** 本文は `--color-dark` を `background: var(--color-bg)` または `var(--color-white)` の上で使い、WCAG を意識する。警告色は背景との組み合わせに注意。

---

## 4. 実装チェックリスト（新規 HTML 用）

1. `:root` に **2 節のコードブロックと同一**のカスタムプロパティを置く（独自の `#xxxxxx` を直書きしない）。
2. `body` は原則 `background: var(--color-bg); color: var(--color-dark);`。
3. カード類の下地は `var(--color-white)`。
4. インタラクションの強調は `accent3`〜`accent5` を軸に、ホバーは隣接トークンまたは `color-mix` で一段変化させる。
5. `:focus-visible` は `outline: 2px solid var(--color-accent4);`（または `accent5`）など、アクセントから選ぶ。
6. 窓口へのリンクを追加する場合は `portal/index.html` のリストに項目を足す。

---

## 5. 変更の扱い

- **パレットやコンセプトを変える場合:** まずこのファイルの `:root` ブロックを更新し、次に `portal/index.html` および既存の各 HTML を揃える。
- **系列色が足りない場合:** まず `accent1`〜`accent6` の割り当てを見直す。どうしても不足する場合のみ、このファイルにトークン追加を提案し、全ツールで共有してから使う。
